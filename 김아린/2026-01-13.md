### MCP란 무엇인가

* **Model Context Protocol (MCP)** 은 LLM 에이전트가 외부 프로그램, 데이터, API와 상호작용할 수 있도록 표준화된 프로토콜.
* MCP를 사용하면 다양한 외부 시스템을 **표준 방식으로 연결**할 수 있고, 한 번 구현해 두면 여러 LLM과 재사용할 수 있음.
### MCP 서버와 MCP 클라이언트

* **MCP 클라이언트**: LLM(AI)이 실제 실행되는 측으로, 요청을 보냄.
* **MCP 서버**: 요청을 받아 실제 기능이나 데이터를 제공하는 쪽.
* 대표적인 MCP 클라이언트 예: Cursor, Claude Desktop 등.

### 해커톤 진행

1. **참가 동기와 구성**

   * 2025년 4월 사내 채팅에서 시작되어 약 8명이 모여 진행. 
2. **목표**

   * Cursor IDE에서 프롬프트를 통해 **배달의민족 장보기 쇼핑 서비스 API** 기능을 사용하는 MCP 서버 구현.
3. **MCP 서버 구현**

   * Python 기반 MCP 서버를 코드로 작성하고, 쇼핑 API에서 데이터를 조회하는 도구(tool)를 정의함.
4. **흐름**

   * 서버 부팅 → 클라이언트 list_tools 호출 → LLM이 적절한 tool 선택 → call_tool로 서버 실행 → 결과 반환.

### 실전 경험

* Python MCP 서버 예제는 기본적인 tool 정의와 STDIO 입출력 방식으로 구현됨.
* 실제 서비스 연동을 위해 API 명세 등을 활용해 함수 구현 및 prompt 설정을 진행함.
* Cursor와 같은 클라이언트가 서버와 통신하며, tool을 통해 상품 조회가 가능한 상태를 확인함.
