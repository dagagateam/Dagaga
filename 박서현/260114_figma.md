# Figma 활용 가이드

## 1. Figma란?

Figma는 **웹 기반 UI/UX 디자인 협업 툴**로, 디자이너·기획자·개발자가 동시에 같은 화면을 보며 작업할 수 있는 것이 가장 큰 장점이다.

- 실시간 협업
- 설치 없이 브라우저 사용 가능
- 디자인 → 개발 연계가 쉬움
- 컴포넌트 기반 설계에 강점

---

## 2. 기본 개념 정리

### 2.1 파일 구조

- **Team**: 프로젝트 단위 조직
- **Project**: 서비스 또는 앱 단위
- **File**: 실제 디자인 작업 파일
- **Page**: 기획 / 디자인 / 컴포넌트 등 구분
- **Frame**: 화면 단위 (웹, 모바일)
- **Layer**: UI 구성 요소

---

### 2.2 주요 UI 개념

| 개념        | 설명                                 |
| ----------- | ------------------------------------ |
| Frame       | 화면 단위 컨테이너                   |
| Group       | 여러 레이어 묶기                     |
| Component   | 재사용 가능한 UI                     |
| Instance    | 컴포넌트의 복사본                    |
| Variant     | 상태별 컴포넌트 (hover, disabled 등) |
| Auto Layout | 자동 정렬 및 반응형 레이아웃         |

---

## 3. 실무에서 중요한 핵심 기능

### 3.1 Auto Layout

- padding, spacing 자동 관리
- 텍스트 길이 변화 대응
- 반응형 UI 구현 가능

**활용 예**

- 버튼
- 리스트 아이템
- 카드 UI
- 채팅 말풍선

---

### 3.2 Component & Variant

- 중복 디자인 방지
- 수정 시 전체 반영
- 상태 관리 용이

**권장 Variant 구성**

- default
- hover
- active
- disabled
- error

---

### 3.3 Styles (디자인 토큰)

- Color Styles
- Text Styles
- Effect Styles

**디자인 시스템의 핵심**

- 색상 변경 시 전체 UI 일괄 수정
- 브랜드 컬러 관리 용이

---

## 4. 협업을 위한 활용법

### 4.1 코멘트 기능

- 특정 UI에 직접 피드백
- @멘션으로 담당자 호출
- 히스토리 관리 가능

---

### 4.2 개발자 핸드오프 (Dev Mode)

- CSS / iOS / Android 코드 자동 제공
- Padding, margin, font 정보 확인
- 에셋(svg, png) 바로 추출

**개발자에게 좋은 점**

- 명세서 감소
- 커뮤니케이션 비용 절감

---

## 5. 기획자를 위한 활용 팁

### 5.1 와이어프레임

- 회색톤, 단순 박스 구조
- 기능 흐름 중심 설계

### 5.2 프로토타입

- 화면 간 연결
- 클릭 / hover / drag 이벤트
- 실제 앱처럼 시연 가능

---

## 6. 디자이너 실무 팁

- Frame 이름 규칙 통일
- Layer 정리 습관화
- 컴포넌트는 초기에 설계
- Auto Layout은 최대한 적극 사용
- 텍스트는 반드시 Text Style로 관리

---

## 7. 개발자 협업 팁

- px 단위 맹신 X → 구조 이해 O
- Auto Layout 구조 확인
- Variant 상태 체크
- 공통 컴포넌트 기준으로 개발
- Figma 기준을 “정답”으로 두기

## 9. Figma 사용 시 자주 하는 실수

- 컴포넌트 없이 복붙
- Auto Layout 미사용
- Layer 이름 미정리
- 디자인과 개발 기준 불일치
- 색상/폰트 스타일 미통일

---

## 10. Figma 잘 쓰는 사람의 특징

- 구조를 먼저 생각한다
- 반복되는 UI를 싫어한다
- 협업을 전제로 디자인한다
- 개발을 고려한 설계를 한다
