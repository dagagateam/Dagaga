# Figma란 무엇이며 웹사이트 제작에 어떻게 사용하는가


## 1. Figma란 무엇인가

Figma는 웹 기반 UI/UX 디자인 협업 도구로 웹사이트와 애플리케이션의 화면을 설계하고 팀원들과 실시간으로 협업할 수 있도록 도와주는 것

웹사이트를 만들 때 Figma는  **코딩 전에 화면을 설계하는 단계**에서 사용되며
기획 → 디자인(Figma) → 개발 → 배포 중 디자인 단계의 핵심 도구로 사용

설치형 프로그램이 아니라 브라우저에서 바로 사용할 수 있다는 점이 큰 특징

---

## 2. Figma를 사용하는 이유

- 디자이너와 개발자의 협업 용이
- 실시간 공동 작업 가능
- 버전 관리 자동화
- 링크 공유만으로 피드백 가능
- 디자인 결과물을 코드로 옮기기 쉬움

---


## 3. Figma 기본 개념

### 3.1 Frame
- 하나의 화면 단위
- 예: 로그인 페이지, 메인 페이지
- 웹, 모바일 해상도 기준 제공

---

### 3.2 Component
- 버튼, 헤더, 카드 등 재사용 가능한 UI 요소
- 한 번 수정하면 모든 사용처에 반영됨

---

### 3.3 Auto Layout
- 요소 간 간격과 정렬을 자동 관리
- 반응형 웹 디자인에 매우 중요

---

### 3.4 Styles
- 색상, 글꼴, 효과를 스타일로 관리
- 디자인 통일성 유지

---

## 4. 웹사이트 제작 단계별 Figma 사용법

---

### 4.1 와이어프레임 제작

목적  
- 화면 구조와 배치만 설계
- 디자인보다 기능 흐름에 집중

작업 내용  
- 박스, 텍스트 중심 구성
- 버튼 위치, 영역 구분

---

### 4.2 UI 디자인

목적  
- 실제 서비스와 유사한 화면 제작

작업 내용  
- 색상, 폰트 적용
- 컴포넌트 생성
- Auto Layout 적용

---

### 4.3 프로토타입 제작

목적  
- 화면 전환 흐름 시각화

작업 내용  
- 버튼 클릭 시 페이지 이동
- 팝업, 모달 시뮬레이션

---

### 4.4 개발자 전달 (핸드오프)

개발자는 Figma에서 다음 정보를 확인한다.

- 색상 코드
- 폰트 정보
- 간격, 크기
- 이미지 및 아이콘 다운로드
- CSS 스타일 참고

---

## 5. Figma 활용법
### 개발자 관점에서

- 디자인을 기준으로 HTML 구조 설계
- 컴포넌트를 React/Vue 컴포넌트로 분리
- Auto Layout을 기준으로 Flex/Grid 적용
- 디자인 수치 그대로 반영

### 팀 프로젝트에서

- 페이지별 Frame 이름 명확히 하기
- 컴포넌트 네이밍 규칙 통일
- 코멘트 기능으로 피드백 남기기
- 디자인 확정 후 수정 범위 명확히 하기
