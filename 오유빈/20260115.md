# JIRA 개념과 기본 사용법 정리

## 1. JIRA란 무엇인가

JIRA는 Atlassian에서 개발한 프로젝트 관리 및 이슈 트래킹 도구

주로 소프트웨어 개발 팀에서 업무 진행 상황을 관리하고 협업하기 위해 사용

---

## 2. JIRA의 핵심 목적

- 할 일(Task)을 체계적으로 관리
- 개발 이슈 및 버그 추적
- 팀원 간 업무 진행 상황 공유
- 일정과 우선순위 관리
- 애자일(Scrum, Kanban) 방식 지원

---

## 3. JIRA의 주요 개념

### 3.1 프로젝트(Project)
- 하나의 서비스 또는 작업 단위
- 예: 웹 개발 프로젝트, 앱 개발 프로젝트

---

### 3.2 이슈(Issue)
- JIRA에서 관리하는 모든 작업 단위
- 예: 기능 개발, 버그 수정, 문서 작업

이슈 타입 예시
- Epic
- Story
- Task
- Bug

---

### 3.3 워크플로우(Workflow)
- 이슈가 진행되는 상태 흐름

기본 예시
- To Do
- In Progress
- Done

---

### 3.4 스프린트(Sprint)
- 일정 기간 동안 수행하는 작업 묶음
- 보통 1~2주 단위

---

### 3.5 백로그(Backlog)
- 아직 진행되지 않은 이슈 목록
- 앞으로 해야 할 작업 리스트

---

## 4. JIRA 기본 사용 흐름

### 4.1 프로젝트 생성
1. JIRA 로그인
2. 프로젝트 생성 클릭
3. 프로젝트 유형 선택
   - Scrum
   - Kanban
4. 프로젝트 이름 설정

---

### 4.2 이슈 생성
1. Create 버튼 클릭
2. 이슈 타입 선택 (Story, Task, Bug 등)
3. 제목 및 설명 작성
4. 담당자(Assignee) 지정
5. 우선순위 설정

---

### 4.3 이슈 상태 변경
- 작업 진행에 따라 상태 변경
  - To Do → In Progress → Done

드래그 앤 드롭으로 보드에서 이동 가능

---

### 4.4 스프린트 운영 (Scrum 기준)
1. 백로그에 이슈 등록
2. 스프린트 시작
3. 스프린트 기간 동안 작업 진행
4. 스프린트 종료 및 리뷰

---

## 5. JIRA 보드(Board) 종류

### 5.1 Scrum Board
- 스프린트 단위로 작업 관리
- 계획 → 개발 → 리뷰에 적합

---

### 5.2 Kanban Board
- 흐름 중심 작업 관리
- 지속적인 업무 처리에 적합

---

## 6. JIRA 기본 사용 예시 (팀 프로젝트)

- Epic: 사용자 인증 기능
  - Story: 회원가입 페이지 구현
  - Task: 이메일 인증 로직 개발
  - Bug: 로그인 오류 수정

---

## 7. JIRA 사용 시 장점

- 업무 가시성 향상
- 협업 효율 증가
- 일정 관리 용이
- 역할 분담 명확

---

## 8. JIRA 사용 시 주의사항

- 이슈를 너무 세분화하지 않기
- 상태 변경을 자주 반영하기
- 담당자 미지정 이슈 방치하지 않기
