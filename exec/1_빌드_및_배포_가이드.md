## 2. 배포 아키텍처

본 프로젝트는 **AWS EC2** 단일 인스턴스에 **Docker** 기반으로 배포됩니다.
Jenkins가 GitLab Webhook을 통해 코드를 빌드하고, 운영 서버에서 도커 컨테이너를 실행합니다.
**System Nginx**가 리버스 프록시 및 SSL 처리를 담당하며, 내부 포트로 트래픽을 라우팅합니다.

### 포트 구성
*   **443 (HTTPS)**: System Nginx (Entrypoint)
*   **8080**: Backend Container (Spring Boot)
*   **3000**: Frontend Container (Nginx serving React)
*   **9090**: Jenkins Container
*   **6379**: Redis Container

<br>

## 3. 환경 변수 설정

배포 시 `docker-compose.yml`에서 참조하는 `.env` 파일과 Jenkins 설정이 필요합니다.

### 3-1. 주요 설정 파일 목록
DB 접속 정보 및 외부 서비스 키가 정의된 파일들은 다음과 같습니다.
1.  **Backend Env**: `backend/dagaga/.env` (DB, Redis, JWT, API Keys)
2.  **Frontend Env**: `frontend/dagaga/.env` (API URL)
3.  **Nginx Config**: `/etc/nginx/sites-available/i14b110.p.ssafy.io` (Reverse Proxy, SSL)

### 3-2. Backend `.env` (`backend/dagaga/.env`)
서버의 `/home/ubuntu/.env` 경로에 위치해야 하며, 백엔드 컨테이너에 마운트됩니다.

```properties
# Database
DB_HOST=your-rds-endpoint.amazonaws.com
DB_PORT=your-rds-port
DB_NAME=postgres
DB_USERNAME=root
DB_PASSWORD=your_password
GEMINI_API_KEY=your_gms_api_key
ASR_SERVER=http://jukang.duckdns.org:8000
GMS_SERVER=http://jukang.duckdns.org:7000

# Redis Configuration
REDIS_HOST=localhost
REDIS_PORT=6379

# JWT Configuration
JWT_SECRET=your_jwt_secret
JWT_ACCESS_TOKEN_EXPIRY=1800
JWT_REFRESH_TOKEN_EXPIRY=604800

# Security Configuration
MAX_CONCURRENT_SESSIONS=3

# Frontend URL for OAuth redirects (Production)
FRONTEND_URL=https://i14b110.p.ssafy.io

# Google OAuth Configuration
GOOGLE_CLIENT_ID=your_client_id
GOOGLE_CLIENT_SECRET=your_client_secret

# Email Configuration
MAIL_HOST=smtp.gmail.com
MAIL_PORT=587
MAIL_USERNAME=your_email@gmail.com
MAIL_PASSWORD=your_password
```

### 3-3. Frontend `.env` (`frontend/dagaga/.env`)
빌드 시점에 사용되는 환경 변수입니다.

```properties
# API Base URL (Backend Endpoint)
VITE_API_URL=/api/v1
VITE_BACKEND_URL=https://i14b110.p.ssafy.io
```

## 4. EC2 및 Docker 설정

### 4-1. Docker 설치
```bash
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh
sudo usermod -aG docker $USER
```

### 4-2. Jenkins 설치 (Docker)
`docker-compose-jenkins.yml`을 사용하여 실행합니다.

```yaml
services:
  jenkins:
    image: jenkins/jenkins:lts-jdk17
    ports:
      - "9090:8080"
    volumes:
      - ./jenkins_home:/var/jenkins_home
      - /var/run/docker.sock:/var/run/docker.sock
    user: root
    environment:
      - JENKINS_OPTS=--prefix=/jenkins
```

## 5. Nginx 설정

EC2에 직접 설치된 Nginx(`/etc/nginx/sites-available/i14b110.p.ssafy.io`)가 **SSL Termination**과 **라우팅**을 담당합니다.

```nginx
server {
    server_name i14b110.p.ssafy.io;

    # 1. 백엔드 API
    location /api {
        proxy_pass http://127.0.0.1:8080;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        
        # WebSocket 지원
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }

    # 2. 젠킨스 CI/CD
    # 반드시 jenkins 환경변수에 --prefix=/jenkins 가 설정되어 있어야 합니다.
    location /jenkins {
        proxy_pass http://127.0.0.1:9090/jenkins;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        
        # Jenkins 리다이렉트 문제 해결
        proxy_redirect http:// https://;
        sendfile off;
    }

    # 3. 프론트엔드
    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # --- SSL 설정 ---
    listen 443 ssl;
    ssl_certificate /etc/letsencrypt/live/i14b110.p.ssafy.io/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/i14b110.p.ssafy.io/privkey.pem;
    include /etc/letsencrypt/options-ssl-nginx.conf;
    ssl_dhparam /etc/letsencrypt/ssl-dhparams.pem;
}
```

## 6. 빌드 및 배포 프로세스 (Jenkins)

1.  **GitLab Push**: 코드 변경 사항이 `develop` 브랜치에 푸시됩니다.
2.  **Webhook Trigger**: GitLab Webhook이 Jenkins 빌드를 트리거합니다.
3.  **Build & Push**:
    *   Backend/Frontend Docker 이미지 빌드
    *   Docker Hub로 Push (`your-dockerhub-username/dagaga-backend`, `your-dockerhub-username/dagaga-frontend`)
4.  **Deploy**:
    *   `docker compose down` -> `pull` -> `up` 과정을 통해 컨테이너 최신화
    *   Docker System Prune으로 자원 정리

## + 배포 전 수정해야 할 항목
이 프로젝트를 다른 환경(도메인, DB)에 배포할 경우, 아래 항목들을 반드시 본인의 환경에 맞게 수정해야 합니다.

1.  **도메인 (Domain Name)**
    *   문서 내의 `i14b110.p.ssafy.io`를 본인의 도메인으로 변경해야 합니다.
    *   **대상 파일**: `backend/dagaga/.env` (`FRONTEND_URL`, `VITE_BACKEND_URL`), `nginx/sites-available` 설정 파일

2.  **데이터베이스 (Database)**
    *   본인의 DB 엔드포인트와 인증 정보로 수정해야 합니다.
    *   **대상 파일**: `backend/dagaga/.env` (`DB_HOST`, `DB_USERNAME`, `DB_PASSWORD`)

3.  **외부 API 키 (External API Keys)**
    *   본인이 발급받은 API 키로 교체해야 합니다.
    *   **대상 파일**: `backend/dagaga/.env`
        *   `GEMINI_API_KEY`: Google AI Studio에서 발급
        *   `GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET`: Google Cloud Console에서 발급
        *   `MAIL_PASSWORD`: Gmail 앱 비밀번호 발급

4.  **SSL 인증서 (SSL Certificate)**
    *   본인의 도메인에 맞춰 Certbot을 다시 실행하여 인증서를 발급받아야 합니다.
    *   **명령어**: `sudo certbot --nginx -d [YOUR_DOMAIN]`
